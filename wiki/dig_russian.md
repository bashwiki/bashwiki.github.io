# [리눅스] Bash dig 사용법

## Обзор
Команда `dig` (Domain Information Groper) используется для выполнения DNS-запросов. Она позволяет пользователям получать информацию о DNS-записях, таких как A, AAAA, MX, CNAME и других. `dig` является мощным инструментом для диагностики и отладки проблем с DNS, а также для получения информации о доменах.

## Использование
Основной синтаксис команды `dig` выглядит следующим образом:

```
dig [опции] [домен] [тип записи]
```

### Общие опции:
- `@<сервер>`: Указывает DNS-сервер, к которому будет отправлен запрос. Если не указано, используется сервер, указанный в настройках системы.
- `+short`: Выводит краткий результат, показывающий только запрашиваемую информацию.
- `-t <тип>`: Указывает тип DNS-записи, которую нужно запросить (например, A, AAAA, MX и т.д.).
- `+trace`: Позволяет проследить путь запроса от корневого DNS-сервера до запрашиваемого домена.

## Примеры
### Пример 1: Получение A-записи
Чтобы получить A-запись для домена example.com, выполните следующую команду:

```bash
dig example.com A
```

### Пример 2: Использование краткого вывода
Если вы хотите получить краткий вывод A-записи для того же домена, используйте опцию `+short`:

```bash
dig example.com A +short
```

## Советы
- Используйте опцию `+trace`, если вам нужно отследить, как DNS-запрос проходит через различные серверы. Это может помочь в диагностике проблем с DNS.
- Если вы часто запрашиваете информацию о определенных доменах, рассмотрите возможность создания алиасов в вашем `.bashrc` для упрощения команд.
- Помните, что результаты `dig` могут отличаться в зависимости от используемого DNS-сервера, поэтому при необходимости указывайте конкретный сервер с помощью опции `@`.
