# [리눅스] Bash nohup 사용법

## 概要
`nohup`（ノーハングアップ）は、シェルコマンドを実行する際に、ユーザーがログアウトしてもそのプロセスが継続して実行されるようにするためのコマンドです。主に、長時間実行されるタスクやバックグラウンドで実行する必要があるプロセスに使用されます。`nohup`を使用することで、セッションが切断されてもプロセスが終了することを防ぎます。

## 使用法
`nohup`の基本的な構文は次のとおりです。

```bash
nohup コマンド [オプション] &
```

### 一般的なオプション
- `&`：コマンドをバックグラウンドで実行します。
- `-o`：出力を指定したファイルにリダイレクトします（デフォルトは`nohup.out`）。
- `-h`：ヘルプメッセージを表示します。

## 例
### 例1: バックグラウンドでスクリプトを実行する
次のコマンドは、`my_script.sh`というスクリプトをバックグラウンドで実行し、ログアウトしてもその実行を継続します。

```bash
nohup bash my_script.sh &
```

### 例2: 出力をファイルにリダイレクトする
以下のコマンドは、`my_long_running_command`を実行し、その出力を`output.log`というファイルに保存します。

```bash
nohup my_long_running_command > output.log 2>&1 &
```

## ヒント
- `nohup`を使用する際は、出力をファイルにリダイレクトすることをお勧めします。これにより、実行中のプロセスのログを確認することができます。
- プロセスがバックグラウンドで実行されている場合、`jobs`コマンドを使用してその状態を確認できます。
- `disown`コマンドを使用すると、バックグラウンドプロセスをシェルのジョブリストから削除し、シェルが終了してもプロセスが継続するようにできます。
