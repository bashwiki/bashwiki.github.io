# [리눅스] Bash git 사용법

## 概述
`git` 是一个分布式版本控制系统，用于跟踪计算机文件的变化，特别是源代码文件。它允许多个开发者在同一个项目上协作，记录每一次的修改，并能够轻松地回溯到之前的版本。`git` 的主要目的是提高开发效率和代码管理的灵活性。

## 用法
基本的 `git` 命令语法如下：
```
git [command] [options] [arguments]
```
常见的选项包括：
- `init`：初始化一个新的 Git 仓库。
- `clone`：从远程仓库克隆一个项目。
- `add`：将文件的更改添加到暂存区。
- `commit`：提交暂存区的更改到本地仓库。
- `push`：将本地仓库的更改推送到远程仓库。
- `pull`：从远程仓库拉取并合并更改。

## 示例
1. 初始化一个新的 Git 仓库：
   ```bash
   git init my_project
   ```
   这条命令将在当前目录下创建一个名为 `my_project` 的新目录，并初始化一个新的 Git 仓库。

2. 从远程仓库克隆一个项目：
   ```bash
   git clone https://github.com/username/repo.git
   ```
   这条命令将从指定的 GitHub 仓库克隆代码到本地。

## 提示
- 在提交更改时，务必使用清晰的提交信息，以便其他开发者理解更改的目的。
- 定期使用 `git pull` 命令更新本地仓库，以保持与远程仓库的同步。
- 使用分支（branch）功能来管理不同的开发任务，避免在主分支上直接进行实验性更改。
- 学习使用 `.gitignore` 文件来排除不必要的文件和目录，以保持仓库的整洁。
