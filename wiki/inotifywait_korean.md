# [리눅스] Bash inotifywait 사용법

## Overview
`inotifywait`는 리눅스에서 파일 시스템 이벤트를 모니터링하는 명령어입니다. 이 명령어는 특정 파일이나 디렉토리에서 발생하는 변경 사항(예: 파일 생성, 수정, 삭제 등)을 감지하고, 이러한 이벤트가 발생할 때마다 알림을 제공합니다. 주로 스크립트에서 파일 시스템 변화를 감지하여 자동화 작업을 수행하는 데 사용됩니다.

## Usage
`inotifywait`의 기본 구문은 다음과 같습니다:

```bash
inotifywait [옵션] [파일 또는 디렉토리]
```

### 일반적인 옵션
- `-m`: 지속적으로 모니터링합니다. 이벤트가 발생할 때마다 계속해서 알림을 받습니다.
- `-r`: 하위 디렉토리까지 재귀적으로 모니터링합니다.
- `-e`: 감지할 이벤트를 지정합니다. 예를 들어, `modify`, `create`, `delete` 등의 이벤트를 지정할 수 있습니다.
- `--format`: 출력 형식을 지정합니다. 기본적으로는 간단한 텍스트 형식으로 출력되지만, 사용자 정의 형식을 설정할 수 있습니다.

## Examples
### 예제 1: 특정 디렉토리에서 파일 변경 감지
다음 명령어는 `/path/to/directory` 디렉토리에서 파일이 생성되거나 수정될 때마다 알림을 출력합니다.

```bash
inotifywait -m -e create,modify /path/to/directory
```

### 예제 2: 하위 디렉토리 포함하여 모든 변경 사항 감지
다음 명령어는 `/path/to/directory`와 그 하위 디렉토리에서 모든 파일 시스템 이벤트를 모니터링합니다.

```bash
inotifywait -m -r -e modify,delete /path/to/directory
```

## Tips
- `inotifywait`는 시스템 리소스를 효율적으로 사용하므로, 파일 시스템 이벤트를 실시간으로 감지해야 하는 경우 매우 유용합니다.
- 스크립트와 함께 사용하여 자동화 작업을 설정할 수 있습니다. 예를 들어, 파일이 변경될 때마다 특정 스크립트를 실행하도록 설정할 수 있습니다.
- 모니터링할 이벤트를 명확히 지정하여 불필요한 알림을 줄이는 것이 좋습니다. 필요한 이벤트만 선택하여 설정하세요.
- `--format` 옵션을 사용하여 출력 형식을 사용자 정의하면, 로그 파일에 기록하거나 다른 시스템으로 전송할 때 유용합니다.
