# [리눅스] Bash resize2fs 사용법

## Overview
`resize2fs`는 리눅스에서 ext2, ext3, ext4 파일 시스템의 크기를 조정하는 데 사용되는 명령어입니다. 이 명령어는 파일 시스템의 크기를 늘리거나 줄이는 데 유용하며, 주로 디스크 파티션의 크기를 변경한 후에 파일 시스템을 조정하는 데 사용됩니다. `resize2fs`는 파일 시스템의 데이터 손실 없이 크기를 조정할 수 있도록 설계되었습니다.

## Usage
기본 구문은 다음과 같습니다:

```bash
resize2fs [옵션] <파일시스템|장치>
```

### 일반적인 옵션
- `-f`, `--force`: 파일 시스템이 사용 중일 때도 강제로 크기를 조정합니다.
- `-p`, `--progress`: 크기 조정 진행 상황을 표시합니다.
- `-M`, `--min`: 파일 시스템을 최소 크기로 조정합니다.
- `-s`, `--size`: 지정된 크기로 파일 시스템을 조정합니다.

## Examples
### 예제 1: 파일 시스템 크기 늘리기
다음 명령어는 `/dev/sda1` 장치의 파일 시스템 크기를 최대한으로 늘립니다.

```bash
sudo resize2fs /dev/sda1
```

### 예제 2: 특정 크기로 파일 시스템 조정하기
다음 명령어는 `/dev/sda1` 장치의 파일 시스템을 20GB로 조정합니다.

```bash
sudo resize2fs /dev/sda1 20G
```

## Tips
- `resize2fs`를 사용하기 전에 항상 중요한 데이터의 백업을 수행하는 것이 좋습니다. 파일 시스템 크기를 조정하는 과정에서 데이터 손실이 발생할 수 있습니다.
- 파일 시스템을 조정하기 전에 해당 파일 시스템이 마운트 해제되어 있는지 확인하세요. 마운트된 상태에서 조정할 경우 데이터 손실이 발생할 수 있습니다.
- `resize2fs` 명령어를 실행하기 전에 `df -h` 명령어를 사용하여 현재 파일 시스템의 사용량과 크기를 확인하는 것이 좋습니다.
