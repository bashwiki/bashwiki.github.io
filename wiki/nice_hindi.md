# [리눅스] Bash nice 사용법

## Overview
`nice` एक Bash कमांड है जिसका मुख्य उद्देश्य प्रोग्रामों की प्राथमिकता को नियंत्रित करना है। यह सिस्टम पर चलने वाले विभिन्न प्रक्रियाओं के बीच CPU संसाधनों का वितरण संतुलित करने में मदद करता है। जब आप किसी प्रोग्राम को `nice` के साथ चलाते हैं, तो आप उसे एक विशेष प्राथमिकता (niceness value) प्रदान करते हैं, जो यह निर्धारित करता है कि वह अन्य प्रक्रियाओं की तुलना में कितना CPU समय प्राप्त करेगा। उच्च niceness value वाले प्रोग्राम कम प्राथमिकता के साथ चलते हैं, जबकि निम्न या नकारात्मक value वाले प्रोग्राम को अधिक प्राथमिकता मिलती है।

## Usage
`nice` कमांड का मूल सिंटैक्स निम्नलिखित है:

```bash
nice [OPTION] [COMMAND [ARGUMENTS...]]
```

यहां, `OPTION` में आप प्राथमिकता सेट करने के लिए विभिन्न विकल्पों का उपयोग कर सकते हैं। कुछ सामान्य विकल्प हैं:

- `-n, --adjustment=N`: यह विकल्प आपको niceness value को सेट करने की अनुमति देता है। `N` एक संख्या है जो -20 (उच्च प्राथमिकता) से 19 (कम प्राथमिकता) के बीच हो सकती है। डिफ़ॉल्ट रूप से, यदि आप कोई विकल्प नहीं देते हैं, तो `nice` 10 की मान से चलता है।
- `-h, --help`: इस विकल्प का उपयोग करके आप `nice` कमांड के बारे में मदद प्राप्त कर सकते हैं।
- `-V, --version`: यह विकल्प `nice` कमांड का संस्करण दिखाता है।

## Examples
### उदाहरण 1: एक सामान्य प्रोग्राम को उच्च प्राथमिकता के साथ चलाना
यदि आप एक प्रोग्राम को अधिक प्राथमिकता देना चाहते हैं, तो आप निम्नलिखित कमांड का उपयोग कर सकते हैं:

```bash
nice -n -5 ./my_script.sh
```

इस कमांड में, `my_script.sh` को -5 की niceness value के साथ चलाया जाएगा, जिससे इसे अन्य प्रक्रियाओं की तुलना में अधिक CPU समय मिलेगा।

### उदाहरण 2: एक प्रोग्राम को कम प्राथमिकता के साथ चलाना
यदि आप एक प्रोग्राम को कम प्राथमिकता देना चाहते हैं, तो आप इसे इस तरह से चला सकते हैं:

```bash
nice -n 15 ./my_other_script.sh
```

यहां, `my_other_script.sh` को 15 की niceness value के साथ चलाया जाएगा, जिससे इसे अन्य प्रक्रियाओं की तुलना में कम CPU समय मिलेगा।

## Tips
- `nice` का उपयोग करते समय ध्यान रखें कि यदि आप एक प्रोग्राम को बहुत अधिक प्राथमिकता देते हैं, तो यह अन्य प्रक्रियाओं को प्रभावित कर सकता है, जिससे सिस्टम की समग्र प्रदर्शन पर असर पड़ सकता है।
- यदि आप किसी प्रोग्राम को `nice` के साथ चलाते हैं, तो आप इसे बाद में `renice` कमांड का उपयोग करके प्राथमिकता को बदल सकते हैं।
- `nice` का उपयोग करते समय, यह सुनिश्चित करें कि आप सिस्टम संसाधनों का सही ढंग से प्रबंधन कर रहे हैं, विशेष रूप से जब आप सर्वर या उत्पादन वातावरण में काम कर रहे हों।
