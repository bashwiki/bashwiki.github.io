# [리눅스] Bash builtin 사용법

## Overview
`builtin` कमांड Bash में एक विशेष कमांड है जिसका उपयोग शेल के अंतर्गत उपलब्ध बिल्ट-इन कमांड्स को कॉल करने के लिए किया जाता है। इसका मुख्य उद्देश्य यह सुनिश्चित करना है कि आप किसी विशेष बिल्ट-इन कमांड को शेल के भीतर ही चलाएं, बजाय इसके कि वह किसी बाहरी कमांड के रूप में चलाया जाए। यह उन स्थितियों में विशेष रूप से उपयोगी होता है जब आप यह सुनिश्चित करना चाहते हैं कि आप बिल्ट-इन वर्शन का उपयोग कर रहे हैं, न कि किसी अन्य वर्शन का।

## Usage
`builtin` कमांड का सामान्य सिंटैक्स निम्नलिखित है:

```bash
builtin [command [arguments]]
```

यहाँ `command` वह बिल्ट-इन कमांड है जिसे आप चलाना चाहते हैं, और `arguments` वह अतिरिक्त तर्क हैं जो आप उस कमांड के लिए प्रदान करना चाहते हैं। 

### सामान्य विकल्प:
- कोई विशेष विकल्प नहीं है, लेकिन आप किसी भी बिल्ट-इन कमांड को उसके सामान्य तर्कों के साथ उपयोग कर सकते हैं।

## Examples
### उदाहरण 1: `echo` बिल्ट-इन कमांड का उपयोग
```bash
builtin echo "Hello, World!"
```
इस उदाहरण में, `builtin` का उपयोग करके हम सुनिश्चित कर रहे हैं कि हम Bash के `echo` बिल्ट-इन कमांड को कॉल कर रहे हैं।

### उदाहरण 2: `cd` बिल्ट-इन कमांड का उपयोग
```bash
builtin cd /path/to/directory
```
यह कमांड आपको निर्दिष्ट डायरेक्टरी में ले जाएगा, और यह सुनिश्चित करेगा कि आप Bash के `cd` बिल्ट-इन कमांड का उपयोग कर रहे हैं।

## Tips
- जब आप किसी बिल्ट-इन कमांड का उपयोग कर रहे हों और आपको संदेह हो कि क्या आप सही वर्शन का उपयोग कर रहे हैं, तो `builtin` का उपयोग करें।
- यदि आप किसी विशेष बिल्ट-इन कमांड के व्यवहार को जानना चाहते हैं, तो आप `help` कमांड का उपयोग कर सकते हैं, जैसे `help cd`।
- `builtin` का उपयोग करते समय, ध्यान रखें कि यह केवल बिल्ट-इन कमांड्स के लिए काम करता है, बाहरी कमांड्स के लिए नहीं। 

इस प्रकार, `builtin` कमांड का उपयोग करके आप सुनिश्चित कर सकते हैं कि आप Bash के अंतर्गत उपलब्ध बिल्ट-इन कमांड्स का सही तरीके से उपयोग कर रहे हैं।
